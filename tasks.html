Homework 1: Tasks for de91

Your personal task is to develop a system for marathons and their runners.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2103.hw1.domain.Runner. With the following private fields:

  1. String name
  2. String nationality
  3. String category
  4. int number
Generate necessary getters and setters. [7 marks]

Create class co2103.hw1.domain.Marathon. With the following private fields:

  1. int id
  2. String location
  3. String month
  4. List<Runner> runners
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2103.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Marathon> marathons in class co2103.hw1.Hw1Application. Initialize the list inside class co2103.hw1.Hw1Application to contain one marathon with two runners. [4 marks]

Create controller co2103.hw1.controller.MarathonController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/marathons" should show view marathons/list. [2 marks]
    • Add the list of all marathons from co2103.hw1.Hw1Application to the model as attribute marathons. [2 marks]
    • Create the corresponding JSP to list the location and month of each marathon followed by a link to URL "/runners?marathon={id of marathon}", e.g., "/runners?marathon=3". [4 marks]
  2. Requests for URL "/newMarathon" should show view marathons/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="marathon" and action="/addMarathon" and inputs for all fields of class Marathon except for runners. [5 marks]
  3. POST requests for URL "/addMarathon" should add a new marathon to the list of all marathons in co2103.hw1.Hw1Application and redirect to /marathons. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2103.hw1.controller.RunnerController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/runners" should show view runners/list. [2 marks]
    • Based on request parameter marathon determine the corresponding marathon from co2103.hw1.Hw1Application. [2 marks]
    • Add the list of all runners of that marathon to the model as attribute runners. [2 marks]
    • Create the corresponding JSP for the view to list the name, nationality, category, and number of each runner. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newRunner?marathon={id of marathon whose runners are listed}", e.g., "/newRunner?marathon=3". [4 marks]
  2. Requests for URL "/newRunner" should show view runners/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="runner" and action="/addRunner" and inputs for all fields of class Runner. [6 marks]
    • Also add the request parameter marathon with the ID of the current marathon. [4 marks]
  3. POST requests for URL "/addRunner" should add a new runner to the list of runners of the marathon identified by request parameter marathon and then redirect to /marathons. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Marathon and make it available in controller MarathonController. [4 marks]

When adding a new marathon validate that

  1. The value of field id is unique within the list Hw1Application.marathons. [3 marks]
  2. The value of field location is not empty. [2 marks]
  3. The value of field month is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Runner and make it available in controller RunnerController. [4 marks]

When adding a new runner validate that

  1. The value of field name is not empty. [2 marks]
  2. The value of field nationality is not empty. [2 marks]
  3. The value of field category is either "Professional", "Amateur", or "Under 18s". [3 marks]
  4. The value of field number is between (including) 1 and 500. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
